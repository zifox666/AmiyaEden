# AmiyaEden 权限管理/身份组

## 角色定义

1. SuperAdmin

    - 权限：拥有系统的全部权限，包括用户管理、数据管理、系统设置等。
    - 适用对象：系统管理员，负责整体系统维护和管理。

2. Admin

    - 权限：所有非技术性权限，包括用户管理、数据管理等。

3. Authenticated User

    - 权限：基本访问权限，包括查看数据和使用系统功能。
    - 适用对象：普通用户，已通过身份验证的用户。

4. Guest

    - 权限：有限访问权限，主要用于查看公开信息。
    - 适用对象：未注册或未登录的用户。

5. 军团/联盟组

    - 根据用户名下eve角色所属自动分组，权限由管理员设置，主要用于管理军团/联盟相关的数据访问权限。

## 权限管理

- 权限分配：管理员可以为不同角色分配具体权限，例如数据访问、功能使用等。
- 权限继承：角色之间可以设置权限继承关系，例如 Admin 继承 Authenticated User 的权限。
- 权限检查：系统在用户访问资源或执行操作时，检查用户的角色和权限，确保只有授权用户才能访问特定资源或执行特定操作。
- 角色管理：管理员可以创建、编辑和删除角色，并为角色分配权限。
- 用户角色绑定：用户可以被分配一个或多个角色，系统根据用户的角色来确定其权限范围。
- 军团/联盟权限：根据用户的eve角色所属军团/联盟自动分组，管理员可以为这些组设置特定权限，以控制对军团/联盟相关数据的访问。
- 多租户功能：支持不同军团/联盟的数据隔离和权限管理，确保数据安全和隐私。
- API KEY：保证API KEY可以具体到某个API接口的权限控制，管理员可以为API KEY分配特定权限，以限制其访问范围和功能。
