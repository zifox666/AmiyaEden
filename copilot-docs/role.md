# AmiyaEden 权限结构设计

## 概述

通过组合式权限一个用户可以拥有多个角色，每个角色包含一组权限。用户的最终权限是其所有角色权限的并集。

## 出厂自带角色

1. Administrator：拥有系统的全部权限，能够进行任何操作。
2. SRP: SRP管理员
3. FC: Fleet管理员
4. User: 普通用户，拥有基本的访问权限。

    - 在config.yml设置allow_corp_ids: list[int], 用户下所有角色只要有在其中的corp_id，就授权User。定时根据`server\pkg\eve\esi\task_affiliation.go`进行corp_id权限更新

5. Guest: 访客，权限受限，仅能查看公开信息。

## 功能

1. 在用户管理界面，admin可以进行角色分配
2. 支持按钮权限(user可以访问该页面，但是没有权限操作某些按钮)
3. 角色可以新增和删除，可以在管理页面对路由/按钮进行权限设置
4. 暴露函数接口，允许其他模块对权限进行管理
